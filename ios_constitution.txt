# Project Constitution — Karaoke Companion App (iOS)
> **Non-negotiable.** This document defines the technical and architectural boundaries for the iOS Companion App. All tasks must adhere to these rules to ensure parity with the Android TV Host and Android Companion.

---

## 1. Identity & Scope
iOS Companion App for a local-network karaoke game. Responsibilities:
1. **Discovery & Pairing:** Discover and pair with the Android TV Host via mDNS using `Network.framework`.
2. **Pitch DSP:** Capture microphone audio, detect pitch using the FFT-YIN pipeline, and stream 16-byte binary UDP results.
3. **Song Serving:** Act as a song source by serving local files via a read-only HTTP server with `Range` support.
4. **UI/UX:** Display real-time role status (Singer/Spectator), input levels, and handle the "Active Mic" lifecycle.

## 2. Technology Constraints

| Concern | Mandatory | Forbidden |
| :--- | :--- | :--- |
| **Language** | Swift 6+ (Strict Concurrency) | Objective-C, C++ in UI layers |
| **UI** | SwiftUI | UIKit Storyboards / XIBs |
| **Concurrency** | Swift Concurrency (`async`/`await`, Actors) | `DispatchQueue` (GCD), RxSwift |
| **Audio Capture** | `AVAudioEngine` (`installTap`) | `AVAudioRecorder`, `AudioQueue` |
| **FFT / DSP** | `Accelerate.framework` (vDSP) | Third-party math/FFT libraries |
| **Discovery** | `Network.framework` (`NWBrowser`) | `NSNetServiceBrowser` (Prohibited) |
| **Network Transport**| `NWConnection` (UDP), `URLSessionWebSocketTask` | CocoaAsyncSocket |
| **HTTP Server** | `Swifter` (Pinned to `1.5.0`) | Vapor, GCDWebServer |
| **Storage Access** | `UIDocumentPicker` + `NSFileCoordinator` + Security Bookmarks | Direct path access to external URIs |
| **Persistence** | `UserDefaults` (for path bookmarks/prefs) | CoreData, SwiftData, Realm |
| **Testing** | XCTest + Swift Testing (2024+) | Reflection-based mocking |
| Unit + integration | `XCTest` | Xcode built-in; no external dependency |
| Mocking | Protocol-based test doubles | No third-party mock library; define lightweight `protocol` facades |
| JSON assertions | `JSONDecoder` + `Codable` | Decode and compare field-by-field; never string-compare JSON |

## 3. Architecture: MVVM + Clean Domain
```text
App/
├── Domain/            # Pure Swift — zero UIKit/SwiftUI/AVFoundation imports
│   ├── Models/        # Data classes, immutable (let)
│   ├── UseCases/      # Single-responsibility logic (e.g., StartSinging)
│   └── Repositories/  # Protocols defining data/network/audio boundaries
├── Data/
│   ├── Audio/         # AVAudioEngine wrapper + vDSP Pitch Pipeline
│   ├── Network/       # mDNS Discovery, WebSocket, UDP Datagrams
│   ├── Server/        # Swifter implementation + NSFileCoordinator
│   └── Storage/       # Bookmark resolution and persistence
├── Presentation/      # SwiftUI Views + ViewModels (@Observable)
│   ├── Pairing/       # QR Scanning & Join Logic
│   ├── Singing/       # Active Mic & Score Display
│   └── Lobby/         # Settings & Device Status
└── DI/                # Manual Injection or Factory-based DI
```
**Core Rules:**
* ViewModels must be the only source of truth for the View state.
* All hardware/framework dependencies (`AVAudioEngine`, `NWBrowser`) must be hidden behind protocols in the `Domain` layer.
* Business logic is strictly prohibited in SwiftUI Views.

## 4. Audio & DSP Pipeline
**Zero-Allocation Hot Loops (Normative per §5.2.5):**
* The `installTap` callback (1024 samples @ 44100Hz) must utilize pre-allocated primitive buffers (like  `UnsafeMutableBufferPointer`).
* **Strict Prohibition:** No object instantiation, heap allocation, or string operations inside the audio callback.
* **Pitch Logic:** Perform Voicing Gate → FFT Autocorrelation → Squared Difference → Candidate Selection → 3-frame Median Filter.
* Use `AVAudioSession.Mode.measurement` to bypass OS voice-processing interference.

---

## 5. Networking & Song Serving
* **Discovery**: NWBrowser scanning _karaoke._tcp, wrapped in AsyncStream.
* **Control**: URLSessionWebSocketTask. JSON ≤ 4KB.
* **Audio**: NWConnection UDP. Packet contains binary pitch and timestamp data.
* **File Delivery**: Swifter serving files directly from the user's selected folder via security bookmarks.
* **Security Bookmarks:** Call `startAccessingSecurityScopedResource()` before server access and `stopAccessingSecurityScopedResource()` immediately after the response is sent.
* **Background Limitation:** Set `isIdleTimerDisabled = true` during active sessions to prevent iOS from suspending the HTTP/WebSocket tasks.
* **Binary Format:** UDP `pitchFrame` packets must be exactly 16 bytes and use Little-Endian encoding.

---

## 6. Testing & Quality
* **Coverage:** Minimum 80% line coverage for all modified or new files.
* **Mocking:** Provide a manual `Fake` implementation for every repository protocol in `Data/Testing/`.
* **Logging:** Use Apple's `os.Logger` for structured logging; `print()` is forbidden in production code.
* **Metadata:** `Info.plist` must include `NSLocalNetworkUsageDescription`, `NSBonjourServices`, `NSCameraUsageDescription`, and `NSMicrophoneUsageDescription`.
